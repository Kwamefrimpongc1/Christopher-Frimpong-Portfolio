Enterprise Active Directory Administration Lab

This project demonstrates enterprise Active Directory administration using Windows Server 2022 hosted in Microsoft Azure. The goal was to build a realistic corporate environment that simulates how IT teams manage users, groups, organizational units, permissions, automation, and Group Policy at scale.

Project Objectives

  • Deploy Windows Server 2022 in Microsoft Azure
  • Install and configure Active Directory Domain Services
  • Create a corporate OU structure
  • Implement Role-Based Access Control (RBAC)
  • Create and manage security groups
  • Automate user provisioning with PowerShell and CSV imports
  • Configure and link Group Policy Objects (GPOs)

Environment

  • Microsoft Azure
  • Windows Server 2022
  • Active Directory Domain Services
  • Group Policy Management
  • PowerShell
  • DNS

Organizational Unit Design

corp.local
│
├── Admin
├── SecurityGroups
├── Branch_01
│   ├── Users
│   │   ├── HR
│   │   ├── IT
│   │   └── Finance
│   ├── Devices
│   ├── DisabledUsers
│   └── RetiredDevices
│
└── Branch_02

The OU hierarchy was designed to mirror a real enterprise environment. This structure supports administrative delegation, scalable user management, device organization, and targeted Group Policy deployment.

Role-Based Access Control (RBAC)

Security groups were created to manage permissions using RBAC principles.

  • IT_Admin
  • HR_Staff
  • Finance_Users
  • Workstation_Admins

PowerShell Automation

A PowerShell automation script was developed to create Organizational Units, security groups, and Active Directory user accounts automatically from a CSV file. The script also handled existing users, moved accounts into the correct OUs, and assigned users to the appropriate security groups.

Group Policy Implementation

A workstation administration Group Policy Object was created and linked to the Branch_01 Devices OU. This demonstrates centralized management of workstation permissions and administrative access through Group Policy.
